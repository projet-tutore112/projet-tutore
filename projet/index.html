Le Projet

Contexte & Objectifs

Un CMS de Bureau pour Sites Statiques Zola.

Notre but est de fournir une interface graphique (GUI) locale, packagée sous forme de logiciel installable, permettant de créer et gérer un site Zola existant sans aucune compétence en programmation.

Notre Mission

  • Masquer la complexité technique (Lignes de commande, Markdown strict).
  • Analyser dynamiquement les besoins des thèmes graphiques.
  • Sécuriser le travail des utilisateurs avec un système de versions.
  • Simplifier la mise en ligne avec des déploiements automatisés.

Ce que vous pouvez faire

Actions clés :
  • Créer des pages à partir de templates analysés automatiquement.
  • Ajouter des médias complexes (Vidéos, Carrousels d'images).
  • Naviguer dans l'historique du site et restaurer d'anciennes versions.
  • Prévisualiser le site en temps réel.
  • Publier le site sur un serveur FTP ou via GitHub Pages en 1 clic.

Intelligence du système

L'interface ne se contente pas d'être jolie : elle analyse en temps réel la configuration du site et le code HTML de vos templates pour générer automatiquement les formulaires dont vous avez besoin.

Périmètre Fonctionnel

Édition Avancée (AST)

Formulaires auto-générés, conversion bidirectionnelle intelligente, intégration de composants riches (Carrousels, citations, code).

Déploiement & Versioning

Historique Git transparent pour l'utilisateur, génération du site statique, et publication intégrée (FTP & GitHub Actions).

Logiciel Autonome

Application Desktop indépendante (Windows/Linux/Mac) embarquant toutes les dépendances nécessaires à la création web.
